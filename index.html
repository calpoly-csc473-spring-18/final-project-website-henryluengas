Advanced Rendering

Quarter Project: Raytracer with Video Frame Rendering System and Rudimentary Physics

Throughout the quarter we have been building a raytracer to render static scenes described by povray files. For my final project I altered my raytracer to take any spheres and boxes described in the scene and give them a random horizontal speed while also applying gravity frame by frame and accounting for collisions with the floor and a 20 by 20 bounding box around the origin.

  • Ray Tracer Features:
    • Parses a subset of the povray spec and renders:
      • Point Lights
      • Planes
      • Spheres
      • Triangles
      • Boxes
    • Handles Object Transformations
    • Shadows
    • Local BDRF Lighting/Shading
    • Recursive Raycast for Reflection and Refraction
    • Fresnel Effect
    • Beer's Law
    • Variable Super Sampling AntiAliasing
    • Spacial Data Structures (not working)
    • Global Illumination based on Monte Carlo Ray Tracing
    • Multithreaded Rendering
    • Rudimentary Physics Engine (using transforms)
    • Video Frame Generation System

In order to make the renderer more time efficient for generating videos, I implemented a basic multithreading system to render each pixel of the image plane on a separate thread.

This brought the render time down from 74 seconds on 1 thread (10% utilization), to 4 seconds on 16 threads (99% utilization).
